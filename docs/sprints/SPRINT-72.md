## Pinned rules

**R1 — Norms are served, never constant.** Every diff/prefill/day-equivalent uses the month GET's `dailyNorm` (fraction-scaled, dated); ANNUAL_ACTIVITY null → blank rendering; weekends 0. The ADR-032 D3 prefill ships UNCHANGED: every empty absence cell prefills the DAY's norm on first focus and stays editable — NO full-day snap (D-D owns the real rule later); the handoff's "hele dage" notes render as informational text only.

**R2 — Diff-row semantics (two pinned pro-handoff visible changes).** Diff(day) = (workTime periods + manualHours + absence hours) − served norm. A full-absence day shows **0,0 green** (shipped grid showed −7,4 red); a day with NO registration at all is **BLANK** (shipped grid deliberately showed −norm). Trailing total sums rendered values; the Flex card's month delta and the grid Diff total reconcile from ONE served source (pinned test).

**R3 — Visibility-independence.** ALL computed values (Diff, the interactive row's unallocated remainder, I alt, Sum, the ✓/amber states) compute over ALL served data INCLUDING rows hidden by preferences — visibility filters RENDERING only. A "N skjulte rækker har timer i denne måned" affordance (links to the modal) appears whenever hidden rows carry hours in the viewed month. Pinned test: hide a populated project → remainder/I alt unchanged + affordance shown + employee-approve gate agreement.

**R4 — Preference model.** `user_skema_preferences` container (employee_id PK, initialized_at) + `user_project_selections` gains `sort_order` (legacy-guarded ALTER; backfill from `projects.sort_order` produces duplicates → reads `ORDER BY sort_order, <code>` deterministic tiebreak; the modal's ▲▼ does a dense reindex on write) + new `user_absence_selections (employee_id, absence_type, sort_order)`. Container absent → today's fallback (all org projects / all filtered types); container present → selections authoritative EVEN WHEN EMPTY. Server computes TWO sets: the addable CATALOG = the full filter chain (agreement → org `absence_type_visibility` → eligibility), selection-INDEPENDENT (removed rows stay re-addable); VISIBLE = catalog ∩ selections (stale selections never resurrect org-hidden/ineligible types). `PUT /api/skema/{id}/row-preferences`: self-service (the save endpoint's employee-scope validator), additions validated against the current catalog, container+selections in ONE tx, **plain un-evented rows** per the `ProjectRepository` selection precedent (view preference ≠ domain state; no event family, no audit projection).
