## Problem
The FE‚Üîbackend response **shape** was hand-copied on both sides (`apiClient.get<T>` casts `res.json() as T` with a hand-written `T`; the backend hand-shapes the JSON). A FE list-hook reading a shape the backend doesn't serve passed its own mock test (green) but broke in prod ‚Äî the "fetchEnheder" class, shipped 3√ó (S97/S99/S100). PAT-010 added a regression guard (the backend shape stays stable); it did NOT guarantee FE‚Üîbackend AGREEMENT.

## Pattern ‚Äî the pipeline + 4 gates
**One generated source of truth, gated at every hand-off so drift is a CI failure, not a prod incident.**

```
backend Contracts/ records + .Produces<T>/.Accepts<T>
        ‚îÇ  (--openapi no-DB entrypoint; Swashbuckle, camelCase)
        ‚ñº
   docs/api/openapi.json   ‚îÄ‚îÄ committed
        ‚îÇ  (npm run gen:api; openapi-typescript)
        ‚ñº
 frontend/src/lib/api-types.ts  ‚îÄ‚îÄ committed
        ‚îÇ  (the typed apiClient: get(pathKey, {params,query}))
        ‚ñº
        FE hooks (tsc)
```

**The 4 gates (each closes one hand-off; complementary, not redundant):**
1. **spec‚â°runtime** (`OpenApiSpecRuntimeTests` + `SpecRuntimeMatcher`, xUnit, build-and-test) ‚Äî the committed schema matches the REAL serialized response PER ROUTE: root/array-ness, property presence, camelCase, declared-success-STATUS fidelity (S112), explicit 204, **required-fidelity** (a schema-`required` member absent at runtime = RED; present-but-null SATISFIES required ‚Äî S113) and **enum-fidelity** (a non-null value outside a declared `enum` set = RED; **`required` and `nullable` are ORTHOGONAL claims ‚Äî the enum set never arbitrates null**; null is admissible iff `nullable` ‚Äî S113). `.Produces<T>` is a *convention* that can lie (esp. array-ness ‚Äî author a bare array as `.Produces<IEnumerable<T>>`); THIS is the closer.
2. **spec drift** (`tools/check_openapi_sync.py`, build-and-test ‚Äî needs .NET) ‚Äî the committed `openapi.json` == what `--openapi` regenerates today.
3. **types freshness** (`npm run gen:api && git diff --exit-code`, frontend-build ‚Äî needs Node) ‚Äî the committed `api-types.ts` == what the spec regenerates. (Closes the spec‚ÜíTS hand-off; without it the bug class reappears one level up.)
4. **convention** (`tools/check_openapi_convention.py`, docs job ‚Äî pure Python) ‚Äî every NEW/non-grandfathered operation in `openapi.json` carries a non-empty response (+ requestBody for body verbs) schema, **or declares `204` as its ONLY success response with no content (`.Produces(204)` ‚Äî the typed no-body statement; S112 owner-ratified amendment)**; an untyped `Results.Ok(new {‚Ä¶})` lands with empty content ‚Üí caught; a mixed inferred-empty-200 + 204 op or a no-success-code op still fails (the "only" formulation is load-bearing ‚Äî `--selftest` proves both directions). The grandfather manifest (`tools/openapi-convention-exempt.txt`) only SHRINKS (a stale entry FAILS). **This is the durability keystone** ‚Äî it forces every new endpoint typed while the retrofit drains the existing.

Plus the no-`as` lint (`npm run lint`, frontend-build) ‚Äî the typed-read hooks carry no `as` / no hand-written `get<T>` type arg.

## The paved road (authoring a new admin endpoint ‚Äî S112-amended)
1. Define a named RESPONSE record in `Contracts/`. **Sibling handlers emitting the identical shape share ONE record** (org create/update/move ‚Üí one `OrganizationResponse`) ‚Äî it makes "same wire contract" structural instead of coincidental and halves the strict-types retrofit surface.
2. Return it (`Results.Ok/Created(theRecord)`) + declare `.Produces<T>(code)` with the CORRECT collection-ness (`IEnumerable<T>` for a bare array) AND the code the handler REALLY returns. A body-less mutation: `Results.NoContent()` + `.Produces(204)` ‚Äî no record. **An explicit `.Produces` REPLACES Swashbuckle's inferred empty 200 on an `IResult` handler** (S112-verified), so declared status codes become honest as a side effect of typing. **Keep ONE declared 2xx per endpoint ‚Äî or a HOMOGENEOUS conditional-status set (S115): an op that genuinely returns different 2xx codes from ONE shape (e.g. 201 first-assign / 200 reassign) declares each status with the SAME record; the gate resolves the set IFF every declared 2xx carries one shared `$ref` (204 never a set member; inline schemas rejected conservatively) and asserts the runtime status ‚àà the declared set.** A per-status branch proof requires ONE DEDICATED SEED STATE PER DECLARED STATUS (in separate Organisations for reporting-tree ops), with the actual status equality-asserted before the matcher call ‚Äî set-membership alone cannot prove branch coverage. Shape-copy the HANDLER's anonymous object, never the repo model (enriched projection rows are often deliberate SUBSETS ‚Äî S115's tree row has no `effectiveTo`); and never let a hand-written FE interface `extends` a sibling response type ‚Äî it inherits phantom fields (the S115 `DirectReport` lie, the program's 4th).
3. **Request side: let Swashbuckle INFERENCE speak** ‚Äî the inferred requestBody schema is generated FROM the bound parameter type, truthful by construction. `.Accepts<T>` OVERRIDES inference and can lie exactly like `.Produces`; if it is ever used, it MUST match the bound parameter type. **(S113 correction of the S112 framing: request-body `required` IS emitted where the C# class declares `required` members ‚Äî and that subset is binder-ENFORCED [absence ‚Üí 400]; the S112 sentence "generated request-body types are all-optional" was FALSE ‚Äî S112's own createUser finding rode exactly this mechanism. The honest boundary: only C#-`required` members are strict; non-`required` NRT/value members stay optional in the generated type because the binder tolerates their absence ‚Äî the `ResponseStrictTypesFilter` deliberately does NOT touch request schemas, which would over-claim.)**
4. Regenerate: `dotnet run --project src/Backend/StatsTid.Backend.Api -- --openapi` ‚Üí commit `docs/api/openapi.json`.
5. FE-consumed? `npm run gen:api` ‚Üí commit `api-types.ts`; call via the typed forms ‚Äî `apiClient.get/post/put/delete(pathKey, {params, query, body})` or `apiFetchWithEtag(pathKey, {method, params, ifMatch, body})` for If-Match routes (no `as`, no type args). **Verify by CALL FORM, not compilation: a raw-body/explicit-`T` string-path call SILENTLY falls through to the untyped fallback ‚Äî tsc will NOT flag a missed switch** (S112-verified empirically).
6. Add a per-route spec‚â°runtime assertion (the S112 per-family fixture pattern; a mutation assertion gets its OWN dedicated seeded row ‚Äî xUnit intra-class order is not guaranteed).
7. The 4 gates + the lint enforce all of the above in CI. (Do NOT add the endpoint to the grandfather manifest ‚Äî it only shrinks.)

## Retrofit recipe (Pass ‚â•1, per drained op)
Response record (+`.Produces`, or `.Produces(204)` for body-less) as an EXACT shape-copy ‚Äî deliver a field-mapping table (anonymous‚Üîrecord member) for byte-identity review; NO request-class changes; regen spec+types; switch the FE call-site to the typed form; DELETE the manifest line; add the per-route assertion. The FE switch is audited by call form per route (see step 5). Typed-client overload extensions follow the S112 pattern: typed overloads FIRST constrained to literal path-key unions; explicit-`T`/template-URL callers fall through by constraint/arity; runtime structured-vs-legacy discrimination is key-subset-based and must behavior-coincide on every ambiguous overlap; success-status-aware derivation (200/201 JSON, declared-204 ‚Üí `undefined`, undeclared-content 200s EXCLUDED); proven by synthetic `paths`-shaped fixtures + `@ts-expect-error` tripwires.

## Vindication (S112 + S113, recorded evidence for the program's premise)
The Pass-1 FE switch surfaced the predicted class in NON-test code: **a hand-written FE interface is itself a mock.** Two live prod bugs fell out at compile time ‚Äî `RoleAssignment` claimed `grantedBy`/`grantedAt` (backend: `assignedBy`/`assignedAt`) ‚Üí two admin columns rendered blank in prod; `updateUser` claimed a phantom `username` response field ‚Üí every save silently dropped it from drawer state. Plus three contract lies (phantom fields both directions, an out-of-contract request field, spec-required fields marked optional). **S113 extended the lesson to the NULLABILITY axis: a field-NAME-only drift guard cannot see nullability ‚Äî `RoleAssignment.assignedBy` claimed non-null `string` while the grant path serves `string | null`; the lie survived S111 AND S112 and surfaced only under strict direct assignment.** Convention from the fix: undeclared error payloads (e.g. 412 bodies) narrow via a runtime type GUARD on the no-`as` surface, never a cast.

## Known residuals
**CLOSED by the S113 strict-types phase:** ~~required-strictness~~ and ~~same-name-TYPE-change~~ ‚Äî the `ResponseStrictTypesFilter` emits `required` = all members for every schema in the **response-reachable closure** (transitive from 2xx `.Produces` refs; the truthfulness basis: the null-emitting serializer always writes every record member ‚Äî empirically confirmed against all 26 real per-route responses), so the generated response types are directly strict and the coercion scaffolding (`apiNarrow.ts` / `coerceApiResponse` / `AssertFieldsInSpec`) is DELETED ‚Äî a type flip now fails `tsc` directly. Enum-widening resolved via **`[property: AllowedValues(...)]` on closed-set response discriminators** (unit type / orgType / periodStatus / vikar reason / scopeType ‚Äî each set citing its authority: a DB CHECK or a total projection; OPEN config-keyed sets like `employmentCategory` must NOT be declared) ‚Üí the filter emits `enum: [...]` ‚Üí generated literal unions + the enum-fidelity gate. Exclusions that keep `required` truthful: `[JsonIgnore(WhenWritingNull/Default)]` members (the `VacationSettlementSnapshot` future, test-pinned) and the nullable-`$ref` case below. Overlap policy: a schema reachable from both request and response gets response-truth `required` (over-strict toward the request side, fails SAFE at compile time).
- **The nullable-`$ref` exception (S113; count now 2)** ‚Äî OpenAPI 3.0 cannot express `nullable` on a bare `$ref`, so a CLR-nullable COMPLEX response member marked `required` would claim never-null (the dangerous-direction lie). The filter excludes such members from `required`; the generated type stays optional; the FE consumes optional-as-nullable (normalize `?? null` in-hook for a one-member envelope ‚Äî the S115 `activeVikar` precedent) or overrides the single property (`Omit<Spec,'field'> & { field?: T | null }` ‚Äî the `RosterRow`/`outgoingVikar` precedent). **Members today: 2 (`outgoingVikar`, `activeVikar`). ESCALATION TRIGGER: at 3+, do the `allOf`-wrapper emission + coordinated matcher change** ‚Äî the residual must not multiply silently.
- **The flag-and-defer rule's FIRST FIRING (S115):** `GET /api/admin/employees/{employeeId}/entitlement-eligibility/{entitlementType}` is genuinely POLYMORPHIC (its no-row branch OMITS `effectiveFrom`/`version` KEYS ‚Äî not null-valued), so one record cannot describe both branches and typing it = a wire change (PAT-010-forbidden). It stays grandfathered with an in-manifest comment; a `oneOf`/wire-versioned fix is product work, not retrofit work. The FE keeps ONE sanctioned explicit-T call, lint-narrowed by call FORM (`[arguments.length=2]` esquery ‚Äî tighter than a blanket rule omission).
- **The typed-etag option surface is a compatibility contract (S115):** the structured `apiFetchWithEtag` options are `{method, params, query, ifMatch, ifNoneMatch, body}` ‚Äî `ifNoneMatch` admits ONLY the literal `'*'` (the create-only precondition). Any future option key must be grep-verified absent from legacy `RequestInit` callers before admission (discrimination must never flip). Likewise `SuccessContract`'s single-status surface (constructor + `StatusCode`) is consumed by the S112 injected-lie tests ‚Äî a compatibility contract for future matcher changes.
- **The spec‚â°runtime gate covers only the typed routes** ‚Äî a future typed endpoint that lies passes convention+drift until a per-route assertion is added. Fold into the retrofit. (S112: status fidelity + declared-204; S113: required + enum fidelity; all 26 typed ops carry per-route assertions.)
- **Request-side declaration-vs-binder boundary** (documented, accepted): spec `required` on request bodies = the C#-`required` subset only (binder-enforced); other members stay optional by design.
